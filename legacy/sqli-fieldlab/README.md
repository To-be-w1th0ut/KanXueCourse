# FieldLab SQLi Range

FieldLab 是一个偏实战、偏授课场景的 SQL 注入教学靶场：
- 本地运行，默认只绑定 `127.0.0.1`
- 覆盖 12 个常见 SQL 注入 / 误修复场景
- 每关都有漏洞模式与安全模式对照
- 按“分类学”而不是“题名堆叠”组织教学
- 含教师讲义、学生提示、环境重置脚本

## 一、项目目标

这不是一个“只有 payload 的练习册”，而是一个按分类框架组织的教学平台，帮助学生走通：

`输入点 → SQL 位置 → 生效时机 → 观察方式 → 业务影响 → 修复`

## 二、分类框架

本靶场采用 **1 个主轴 + 4 个副轴** 的组织方法。

### 主轴：按注入位置分类
- 值上下文注入
- 结构上下文注入
- 语句级注入
- 动态 SQL 注入

### 副轴 1：按生效时机分类
- 一次注入
- 二次注入

### 副轴 2：按观察方式分类
- 直接回显型
- 报错型
- 布尔盲注
- 时间盲注
- 带外潜力型（用于讲数据库能力差异）

### 副轴 3：按最终影响分类
- 认证绕过
- 数据读取
- 数据枚举
- 数据篡改
- 语句扩展
- 错误修复对照

### 副轴 4：按数据库方言分类
- MySQL / MariaDB
- PostgreSQL
- SQL Server
- Oracle
- SQLite

## 三、关卡映射

| 关卡 | 主轴 | 观察方式 | 时机 | 影响 |
|---|---|---|---|---|
| L01 登录网关绕过 | 值上下文注入 | 直接回显型 | 一次注入 | 认证绕过 |
| L02 商品检索 UNION 泄露 | 值上下文注入 | 直接回显型 | 一次注入 | 数据读取 |
| L03 工单详情报错注入 | 值上下文注入 | 报错型 | 一次注入 | 数据枚举 |
| L04 门禁查询布尔盲注 | 值上下文注入 | 布尔盲注 | 一次注入 | 数据枚举 |
| L05 物流接口时间盲注 | 值上下文注入 | 时间盲注 | 一次注入 | 数据枚举 |
| L06 审计报表堆叠查询 | 语句级注入 | 直接回显型 | 一次注入 | 语句扩展 |
| L07 二次注入：保存筛选器 | 动态 SQL 注入 | 直接回显型 | 二次注入 | 数据读取 |
| L08 排行榜排序 / LIMIT 注入 | 结构上下文注入 | 直接回显型 | 一次注入 | 数据枚举 |
| L09 成绩调整 UPDATE 注入 | 语句级注入 | 直接回显型 | 一次注入 | 数据篡改 |
| L10 JSON API 参数注入 | 值上下文注入 | 直接回显型 | 一次注入 | 数据读取 |
| L11 伪修复：黑名单过滤 | 值上下文注入 | 直接回显型 | 一次注入 | 错误修复对照 |
| L12 ORM 误用 | 动态 SQL 注入 | 直接回显型 | 一次注入 | 数据读取 |

## 四、技术栈

- Web：Flask
- 数据库：MariaDB 10.11
- 前端：Jinja2 + 自定义教学 UI
- 部署：Docker Compose
- 数据访问：PyMySQL + SQLAlchemy（用于 ORM 误用对照）

## 五、启动方式

### 1. 准备环境变量

```bash
cp .env.example .env
```

### 2. 启动

```bash
docker compose up --build
```

### 3. 访问

- 首页：[http://127.0.0.1:5050](http://127.0.0.1:5050)
- 分类总览：[http://127.0.0.1:5050/labs](http://127.0.0.1:5050/labs)
- 健康检查：[http://127.0.0.1:5050/healthz](http://127.0.0.1:5050/healthz)

## 六、模式说明

每个关卡都支持：
- `?mode=vuln`：漏洞模式
- `?mode=safe`：安全模式

## 七、重置环境

### 全量重置

```bash
./scripts/reset_lab.sh all
```

### 仅重置写操作关卡

```bash
./scripts/reset_lab.sh grade-editor
./scripts/reset_lab.sh report-stacked
./scripts/reset_lab.sh second-order
```

## 八、健康检查

```bash
./scripts/healthcheck.sh
```

## 九、推荐授课顺序

### 阶段 1：先按主轴讲“注入位置”
1. L01 登录网关绕过
2. L02 商品检索 UNION 泄露
3. L08 排行榜排序 / LIMIT 注入
4. L09 成绩调整 UPDATE 注入
5. L07 二次注入：保存筛选器
6. L12 ORM 误用

### 阶段 2：再按观察方式讲“怎么发现成功”
1. L02 直接回显
2. L03 报错型
3. L04 布尔盲注
4. L05 时间盲注

### 阶段 3：最后按业务影响讲“会造成什么”
1. L01 认证绕过
2. L02 / L10 / L12 数据读取
3. L03 / L04 / L05 / L08 数据枚举
4. L09 数据篡改
5. L06 语句扩展
6. L11 错误修复对照
