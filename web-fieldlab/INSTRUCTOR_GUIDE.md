# Unified FieldLab 教师指南

## 一、推荐授课方式

建议按轨道分层授课：
1. SQLi
2. XSS
3. SSTI
4. SSRF
5. 越权
6. CSRF
7. 文件上传
8. 支付逻辑
9. 代码 / 命令注入
10. XXE
11. JSONP
12. 条件竞争

## 二、各轨核心讲法

### 越权
- 先区分未授权访问与已登录后的越权
- 水平越权看 owner 绑定
- 垂直越权看角色与动作级校验

### CSRF
- 核心是“浏览器自动带着受害者身份发请求”
- 先讲无 token 状态修改
- 再讲 Referer / Origin 伪防护
- 再讲 JSON API 与安全配置变更

### 其它轨道
- SQLi：值位 / 结构位 / 动态 SQL
- XSS：上下文 / sink / 传播路径
- SSTI：模板就是代码
- SSRF：谁在发请求、能打到哪里、怎么观测
- 上传：内容、文件名、存储位置、返回方式
- 支付：服务端重算、幂等、原子性
- 注入：解释器边界
- XXE：解析器配置
- JSONP：callback 是代码位
- Race：共享状态与非原子更新

## 三、推荐演示顺序
- 先讲 SQLi / XSS 这两条基础轨
- 再讲 SSTI / SSRF / XXE / 注入
- 然后讲授权与 CSRF
- 最后收束到上传、支付和条件竞争

## 四、统一修复原则
- 不可信输入不要进入解释器 / 模板 / shell / 实体解析器
- 浏览器端自动带身份的请求必须配 CSRF 防护
- 未授权、越权、CSRF 都属于“身份与状态边界”问题，但层次不同
- 值位参数化、结构位白名单、状态变更讲事务 / 幂等 / 原子性
