# Unified FieldLab 教师指南

## 一、推荐授课方式

不要一上来混讲所有漏洞，建议按漏洞域分轨授课：
1. SQLi
2. XSS
3. SSTI
4. SSRF
5. 越权

讲完单域后，再做跨域对照：
- SQLi 的“值 vs 结构” 类比 XSS 的“上下文 vs sink”
- XSS 的“二次触发” 类比 SSTI 的“存储后二次渲染”
- SSRF 的“服务端代请求” 对比越权里的“后端信任边界”

## 二、各轨核心讲法

### SQLi
- 先讲注入位置
- 再讲观察方式
- 最后讲修复组合拳

### XSS
- 先讲浏览器上下文
- 再讲 sink
- 最后讲反射/存储/DOM/二次路径

### SSTI
- 强调模板就是代码
- 区分“变量值进入模板”和“用户控制模板源码”
- 讲清数据库里的模板为何仍然危险

### SSRF
- 先问“谁在发请求”
- 再讲 URL 解析差异与重定向链
- 最后讲盲 SSRF 的副作用观测

### 越权
- 先区分“已登录”和“有权访问该对象/动作”
- 水平越权重点讲 owner 绑定
- 垂直越权重点讲后端鉴权而非隐藏按钮

## 三、推荐演示顺序

### 90~120 分钟课
1. SQLi：L01 / L02 / L03 / L08 / L09
2. XSS：L01 / L03 / L04 / L06 / L11
3. SSTI：L01 / L03
4. SSRF：L01 / L02 / L03 / L04
5. 越权：L01 / L02 / L03 / L04

## 四、统一修复原则
- 不可信输入不要进入解释器 / 解析器 / 动态模板
- 值位用参数化，结构位用白名单
- DOM 优先 textContent / createElement
- URL 做协议、主机、网段限制
- 模板不要把用户输入当源码
- 授权校验要在后端、对象级、动作级分别落实
