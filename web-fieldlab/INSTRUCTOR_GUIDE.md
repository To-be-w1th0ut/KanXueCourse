# Unified FieldLab 教师指南

## 一、推荐授课方式

建议按轨道分层授课，不要一上来混讲：
1. SQLi
2. XSS
3. SSTI
4. SSRF
5. 越权
6. 文件上传
7. 支付逻辑
8. 代码 / 命令注入
9. XXE
10. JSONP
11. 条件竞争

讲完单域后，再做跨域对照：
- SQLi 的“值 / 结构位” 类比 XSS 的“上下文 / sink”
- XSS 的“存储后二次触发” 类比 SSTI / XXE 的“存储后二次解析”
- SSRF 与 XXE 都能通向内网目标
- 越权、支付逻辑与条件竞争都属于“状态与规则失守”

## 二、各轨核心讲法

### SQLi
- 先讲注入位置
- 再讲观察方式
- 最后讲修复组合拳

### XSS
- 先讲浏览器上下文
- 再讲 sink
- 最后讲反射 / 存储 / DOM / 二次路径

### SSTI
- 模板就是代码
- 变量值进入模板 ≠ 用户控制模板源码
- 数据库里的模板仍然不可信

### SSRF
- 先问“谁在发请求”
- 再讲解析差异与重定向链
- 最后讲盲 SSRF 的副作用观测

### 越权
- 区分“已登录”与“有权访问该对象 / 动作”
- 水平越权看 owner 绑定
- 垂直越权看后端角色校验

### 文件上传
- 文件内容、文件名、存储位置、响应头分别讲
- 公开同源访问是高风险放大器
- MIME 与扩展名不要只信客户端

### 支付逻辑
- 服务端永远自己算钱
- 优惠券 / 回调 / 库存 / 余额都要看状态机
- 幂等和原子性是重点

### 代码 / 命令注入
- 把 Python 解释器和 shell 都当“解释器边界”讲
- eval / exec / shell=True 一脉相承
- 白名单 DSL 与参数数组分别是两类修法

### XXE
- 重点是解析器配置
- 文件读取与内网请求都只是实体扩展的结果
- 存储后二次解析要单独强调

### JSONP
- JSONP 返回的是 JavaScript，不是 JSON
- callback 参数本质是代码位置
- 敏感接口不应提供 JSONP

### 条件竞争
- 核心是共享状态与非原子更新
- 券、库存、余额、席位只是四种不同业务外观
- 用并发模拟按钮让学生直观看到异常 success_count

## 四、统一修复原则
- 不可信输入不要进入解释器 / 模板引擎 / shell / XML 实体解析器
- 值位用参数化，结构位用白名单
- DOM 优先 textContent / createElement
- URL 做协议、主机、网段限制
- 支付与竞态问题重点看状态机、幂等、事务、原子性
- 授权校验必须在后端、对象级、动作级分别落实
- 文件上传要同时控制类型、名称、路径和返回方式
