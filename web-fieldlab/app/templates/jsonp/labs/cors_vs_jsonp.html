v2 收紧 CORS / v1 JSONP 仍在线

v2：/api/v2/orders        — Access-Control-Allow-Origin 仅白名单（fetch 跨域被拒）
v1：/api/jsonp/v1/orders  — vuln 模式仍可 callback 旁路；safe 模式直接 410。

直接打开对比：
  v2 同源：     /api/v2/orders
  v1 vuln：    /api/jsonp/v1/orders?callback=leak&mode=vuln
  v1 safe：    /api/jsonp/v1/orders?callback=leak&mode=safe

"修了 CORS 没修 JSONP" 的真实危害：
  攻击者写 <script src="http://target/api/jsonp/v1/orders?callback=leak">
  v2 的 CORS 白名单完全不参与；浏览器照样把 v1 响应当 JS 执行。

排查建议

  • 每次 API 迁移都要全量 grep callback、jsonp、application/javascript 三类关键字。
  • 把"下线旧端点"列入迁移 checklist 的强制项。
  • 边缘代理上加规则：任何带 ?callback= 的请求统一 410。
